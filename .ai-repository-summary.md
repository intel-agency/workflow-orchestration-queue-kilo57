## Overview

workflow-orchestration-queue (OS-APOW) is a headless agentic orchestration platform that transforms GitHub Issues into automated execution orders. It uses a 4-pillar architecture (Ear/State/Brain/Hands) to detect work orders, claim tasks, and execute agentic workflows in isolated DevContainers.

## Key Components

### 1. Sentinel Orchestrator (`src/orchestrator_sentinel.py`)
- Persistent polling daemon (60s interval)
- Distributed locking via assign-then-verify pattern
- Shell-Bridge dispatch for worker lifecycle
- Heartbeat posting during long-running tasks

### 2. Notifier Service (`src/notifier_service.py`)
- FastAPI webhook receiver
- HMAC SHA256 signature validation
- Intelligent event triaging
- WorkItem manifest generation

### 3. GitHub Queue (`src/queue/github_queue.py`)
- GitHub Issues as work queue
- Labels as state machine
- Connection pooling with HTTPX
- Provider-agnostic interface (ITaskQueue)

### 4. Data Models (`src/models/work_item.py`)
- Unified WorkItem model
- TaskType and WorkItemStatus enums
- Credential scrubbing utility

## Quick Start

```bash
# Install dependencies
uv sync

# Run tests
uv run pytest

# Run linter
uv run ruff check src/

# Start notifier service (development)
uv run uvicorn src.notifier_service:app --reload

# Start sentinel (requires env vars)
uv run python -m src.orchestrator_sentinel
```
